Introduction:

This project is to classify and predict the ranks of restaurants based on the customers' review and characteristics of restaurants. The purpose is to get the weight of each characteristics and use them to classify the restaurants into multiple ranks in order to recommend for customers. Furthermore, from the result, we can gain the predicted ratings based on their values of characteristics.
As early as 2017, the content of online reviews on sites like Yelp, Glassdoor, Trip Advisor, and even Facebook can mean the difference between success and failure if you are in the restaurant or hotel business. Nowadays, as restaurants search engine and rating applications reinvent themselves, the importance of online reviews can no longer be overlooked because they carry a great deal of weight when diners are looking to eat out. According to the new TripAdvisor “Influences on Diner Decision-Making” survey, 94% of US diners are influenced by online reviews. Since the decision-making process of finding restaurants is heavily influenced by online reviews, it is meaningful to find the weight of each influencing factor in determining the final rate of restaurants.
Our optimization problem is based on customers’ ratings of restaurants, determining how important each characteristic of restaurants is and predicting from the pattern. Our data is from UCI Machine Learning Repository, and it includes the final rating, different characteristics such as alcohol serving and price range. We intend to determine the weight of each factor to see their significance toward the final rating.
The first task is to process the data. After processing the data, the logistic regression is used to classify the restaurants into multiple rating levels. Logistic regression is an approach to modelling the relationship between a dependent variable and one or more independent variables. In our case, it is the model to figure out the relationship between final rating and the other characteristics. We are using gradient descent algorithm to determine the weight which is an iterative optimization algorithm to find the minimum of a function. This study also aims to predict customer satisfaction, which is represented by the overall customer satisfaction level. Overall customer satisfaction is driven by customer perception toward the performance of dining satisfaction factors.
Data:
The original data set profile we need are [rating_final.csv], [chefmozparking.csv]. [chefmozaccepts.csv] and [geoplaces2.csv]. [rating_final.csv] involves the rating, food rating and service rating. Each review occupies one row of the document and in most cases, one person writes multiple reviews over different restaurants. We take the average of the reviews of each restaurants and get a new data set on the final rating of restaurants. All of these actions are done under excel. [chefmozparking.csv] contains the parking status of each restaurants. [chefmozaccepts.csv] includes the cash acceptance and card acceptance of each restaurants. If the restaurant accepts cash, it is set to be 1, otherwise 0. Though it specifies the types of card such as VISA, MasterCard, Discover and many other card payment types, we conclude all of them into the card and set "card" as 1 if any of the card type is accepted. [geoplaces2.csv] includes the other 9 characteristics of the restaurants. The final data set is under the name [avg_rate.csv].

This is the stage to pre-process the data writing data in matrix for further calculation. It determines the data used in each classification step. There are three classification steps, which can divide the data into four ranks based on the average rate: less than 1.0, 1.0 to 1.25, 1.25 to 1.5, and greater than 1.5. Mathematical Model

Logistic regression
Logistic regression is a statistical model used to model the probability of a certain class. This model uses a logistic function to model a binary dependent variable with two possible values represented by an indicator variable, which in our problem is 1 and -1. The corresponding probability can be calculated for each variable based on the logistic function. The threshold value is 0.5. When possibility is larger than 0.5, the variable can be considered as 1; and if less than 0.5, it is -1.

Gradient Descent Solver
To determine the minimum of the above equation, it is required to use the gradient descent solver. A gradient descent solver is a first-order iterative optimization algorithm for finding the minimum of a differentiable function. The way to find the local minimum is to calculate the gradient of the current point and take steps proportional to the negative of the gradient. Thus, the current point becomes closer to the local minimum of the differentiable function. When the step change is small enough, the current point can be considered as the local minimum of this function.

The function below is the main function uses logical regression and gradient descent solver to classify the whole data into four rating categories and find the weights. Then the data is trained three times with the first 75% data as train set and the remaining data as test, the last 75% data as train set and the remaining data as test, the middle 75% data as train set and the remaining data as test.

[ranking.m]
function weights_tot = ranking
% This function uses logical regression and gradient descent solver
% to divide the whole data
% we just classify three time and the data are divided into 4 ranks based on their average rate:
% A. (+inf, 1.5] B. (1.5,1.25] C (1.25,1.0] D (1.0,0]
fprintf('The following is the 4 ranks classification:\n\n');
[Rate_tot,~,place] = data;
Rate_tot = changescale(Rate_tot); % changing the data range using the function changescale
weights_tot =zeros(12,3); % record the weights and possible to check
P_tot= zeros(130,3);
for z = 1:3 % determine the data used for each ranking
clear Rate_use;
if z == 1
Rate_use = Rate_tot(place(1):place(2)-1,:);
fprintf('In the 1st classification:\n');
end
if z == 2
Rate_use = Rate_tot(place(2):place(3)-1,:);
fprintf('In the 2nd classification:\n');
end
if z == 3
Rate_use = Rate_tot(place(3):end,:);
fprintf('In the 3rd classification:\n');
end
[row,~] = size(Rate_use);
train_row = int32(0.75*row);
weights_max = zeros(12,3);
correction_max = zeros(1,3);
max_num = 0;
for j = 1:3
% the following is three types of division of train set and test set
if j == 1
% Use the first 75% data as train set and the remaining data as test
Xtrain = Rate_use(1:train_row,5:end);
Xtest = Rate_use(train_row+1:end,5:end);
ytrain = Rate_use(1:train_row,2);
ytest = Rate_use(train_row+1:end,2);
end
if j == 2
%Use the last 75% data as train set and the remaining data as test
Xtrain = Rate_use(row-train_row+1:end,5:end);
Xtest = Rate_use(1:row-train_row,5:end);
ytrain = Rate_use(row-train_row+1:end,2);
ytest = Rate_use(1:row-train_row,2);
end
if j== 3
%Use the middle 75% data as train set and the remaining data as test
Xtrain = Rate_use(1:int32(0.375*row),5:end);
Xtrain = [Xtrain;Rate_use(int32(0.625*row)+1:end,5:end)];
Xtest = Rate_use(int32(0.375*row)+1:int32(0.625*row),5:end);
ytrain = Rate_use(1:int32(0.375*row),2);
ytrain = [ytrain;Rate_use(int32(0.625*row)+1:end,2)];
ytest = Rate_use(int32(0.375*row)+1:int32(0.625*row),2);
end
%pre process data
Xtrain = log(Xtrain+0.1);
Xtest = log(Xtest+0.1);
weights = zeros(12,1);
threshold = 10^-10; %the minimum increment for stopping criteria.
max = 10^5; %maximum iteration
count = 0;
%Gradient descent solver
while (count<max)
count = count+1;
v = -sum(ytrain.*Xtrain./(1+exp(ytrain.*(Xtrain*weights))))./size(Xtrain,1); %v is the gradient
v_new = -v'; % negative of the gradient is the direction to move
if (abs(0.01 * sum(v_new))<threshold) % stopping criteria: the change less than threshold
break
end
weights = weights+0.01*v_new; % the w moves by step size 0.01* direction
end
weights_max(:,j) = weights;
EE = 0; %predict as 1 and real is 1
ES = 0; %predict as 1 and real is -1
SE = 0; %predict as -1 and real is 1
SS = 0; %predict as -1 and real is -1
% logistic regression to divide the whole data by the
% possiblity
for i = 1:size(Xtest,1)
p = 1./(1+exp(-Xtest(i,:)*weights)); % calculating the possibility
P_tot(i,z)= p;
% EE and SS means the LR is correct, and ES and SE is the
% wrong condition
if p > 0.5
if (ytest(i)==1)
EE = EE + 1;
else
ES = ES + 1;
end
else
if (ytest(i)==1)
SE = SE + 1;
else
SS = SS + 1;
end
end
end
correct_rate = (EE+SS)/size(Xtest,1)*100;
% print the correct rate with different train set and find the
% best weight vector
if j == 1
fprintf('The first 75%% data as train set gets %f percent correct\n', correct_rate);
end
if j == 2
fprintf('The last 75%% data as train set gets %f percent correct\n', correct_rate);
end
if j == 3
fprintf('The middle 75%% data as train set gets %f percent correct\n', correct_rate);
end
correction_max(1,j) = correct_rate;
% finding the max correct rate and find its weight vector
if correction_max(1,1) >= correction_max(1,2)
if correction_max(1,1) >= correction_max(1,3)
max_num = 1;
else
max_num = 3;
end
else
if correction_max(1,2) >= correction_max(1,3)
max_num = 2;
else
max_num = 3;
end
end
end
weights_tot(:,z) = weights_max(:,max_num);
fprintf('\n');
end
end

function weights_tot = ranking_food_service
% This function uses logical regression and gradient descent solver
% to divide the whole data
% we just classify three time and the data are divided into 4 ranks based on their average rate:
% A. (+inf, 1.5] B. (1.5,1.25] C (1.25,1.0] D (1.0,0]
fprintf('The following is the 4 ranks classification:\n\n');
[Rate_tot,~,place] = data;
Rate_tot = changescale(Rate_tot); % changing the data range using the function changescale
weights_tot =zeros(14,3); % record the weights and possible to check
P_tot= zeros(130,3);
for z = 1:3 % determine the data used for each ranking
clear Rate_use;
if z == 1
Rate_use = Rate_tot(place(1):place(2)-1,:);
fprintf('In the 1st classification:\n');
end
if z == 2
Rate_use = Rate_tot(place(2):place(3)-1,:);
fprintf('In the 2nd classification:\n');
end
if z == 3
Rate_use = Rate_tot(place(3):end,:);
fprintf('In the 3rd classification:\n');
end
[row,~] = size(Rate_use);
train_row = int32(0.75*row);
weights_max = zeros(14,3);
correction_max = zeros(1,3);
max_num = 0;
for j = 1:3
% the following is three types of division of train set and test set
if j == 1
% Use the first 75% data as train set and the remaining data as test
Xtrain = Rate_use(1:train_row,3:end);
Xtest = Rate_use(train_row+1:end,3:end);
ytrain = Rate_use(1:train_row,2);
ytest = Rate_use(train_row+1:end,2);
end
if j == 2
%Use the last 75% data as train set and the remaining data as test
Xtrain = Rate_use(row-train_row+1:end,3:end);
Xtest = Rate_use(1:row-train_row,3:end);
ytrain = Rate_use(row-train_row+1:end,2);
ytest = Rate_use(1:row-train_row,2);
end
if j== 3
%Use the middle 75% data as train set and the remaining data as test
Xtrain = Rate_use(1:int32(0.375*row),3:end);
Xtrain = [Xtrain;Rate_use(int32(0.625*row)+1:end,3:end)];
Xtest = Rate_use(int32(0.375*row)+1:int32(0.625*row),3:end);
ytrain = Rate_use(1:int32(0.375*row),2);
ytrain = [ytrain;Rate_use(int32(0.625*row)+1:end,2)];
ytest = Rate_use(int32(0.375*row)+1:int32(0.625*row),2);
end
%pre process data
Xtrain = log(Xtrain+0.1);
Xtest = log(Xtest+0.1);
weights = zeros(14,1);
threshold = 10^-10; %the minimum increment for stopping criteria.
max = 10^5; %maximum iteration
count = 0;
%Gradient descent solver
while (count<max)
count = count+1;
v = -sum(ytrain.*Xtrain./(1+exp(ytrain.*(Xtrain*weights))))./size(Xtrain,1); %v is the gradient
v_new = -v'; % negative of the gradient is the direction to move
if (abs(0.01 * sum(v_new))<threshold) % stopping criteria: the change less than threshold
break
end
weights = weights+0.01*v_new; % the w moves by step size 0.01* direction
end
weights_max(:,j) = weights;
EE = 0; %predict as 1 and real is 1
ES = 0; %predict as 1 and real is -1
SE = 0; %predict as -1 and real is 1
SS = 0; %predict as -1 and real is -1
% logistic regression to divide the whole data by the
% possiblity
for i = 1:size(Xtest,1)
p = 1./(1+exp(-Xtest(i,:)*weights)); % calculating the possibility
P_tot(i,z)= p;
% EE and SS means the LR is correct, and ES and SE is the
% wrong condition
if p > 0.5
if (ytest(i)==1)
EE = EE + 1;
else
ES = ES + 1;
end
else
if (ytest(i)==1)
SE = SE + 1;
else
SS = SS + 1;
end
end
end
correct_rate = (EE+SS)/size(Xtest,1)*100;
% print the correct rate with different train set and find the
% best weight vector
if j == 1
fprintf('The first 75%% data as train set gets %f percent correct\n', correct_rate);
end
if j == 2
fprintf('The last 75%% data as train set gets %f percent correct\n', correct_rate);
end
if j == 3
fprintf('The middle 75%% data as train set gets %f percent correct\n', correct_rate);
end
correction_max(1,j) = correct_rate;
% finding the max correct rate and find its weight vector
if correction_max(1,1) >= correction_max(1,2)
if correction_max(1,1) >= correction_max(1,3)
max_num = 1;
else
max_num = 3;
end
else
if correction_max(1,2) >= correction_max(1,3)
max_num = 2;
else
max_num = 3;
end
end
end
weights_tot(:,z) = weights_max(:,max_num);
fprintf('\n');
end
end

Limitations and Future Work

One limitation of the model is because of the limited data set. Theoretically, the more data we used to train, the more accurate our weight can be. A large data set will result in prediction of restaurants rates with higher accuracy.
One possible future work the model can adapt to is to broaden the area. Our assumption is that in different countries, the weight of different characteristics may vary. Besides reviews, other factors customers may take into consideration when selecting a restaurant might be the restaurant’s mobile presence (or lack thereof), availability of online delivery and the ability to accept reservations, as well as being able to order their food. The assumption is that if the restaurants don’t give diners the flexibility to find the perfect table or meal, they will go elsewhere. These characteristics can be added in the next step of research to better complete the model.
The implementation of this research is quiet explorable. For example,when the initial cost of opening a restaurant is limited, the result and conclusion of this project can help determining which characteristic is worth investigating into in order to put in the smallest amount of money and have the biggest income. Its practicability is obvious when restaurants want to improve from a D level to an A level. From the result, we can tell that the best way is to add more parking spots and it would be better to add valet parking rather than street parking. Also, these D level restaurants can improve their standards by adding accessibility to disabled people. The suggestion of adding parking spots and improving accessibility is based on the positive weights of these characteristics in relation to the final rating from customers. The specific and targeted improvement can avoid unnecessary efforts.

Conclusion

This report develops a mathematical model and solution for determining the optimal operational parameters for a restaurant review. This program could be used in the following ways:
  1. Estimating the restaurants rate based on their characteristics
  2. Determining which characteristics of restaurants are needed for optimal operation.
  3. Consulting for restaurants’ further improvements and a reference for where their efforts should be going.
As the number of classification increases, the accuracy goes down. As in the restaurants rating case, the demand for a high number of classification is not crucial towards the customers’ decision-making process, the classification of two or three categories is recommended for future restaurants rates prediction.
